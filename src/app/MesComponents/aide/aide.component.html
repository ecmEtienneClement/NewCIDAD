APP-POST

App-Post vous permet de recevoir des suggestions de solutions sur vos codes qui ont des erreurs (Bugs).Pour se faire il faudrait enregistré le post sur [Enregistrer Post], le post aura un Titre, Language, Détails et le(s) Code(s).
Régles D'or :
[1]==> Toujour vérifier s'il n'y a pas un post similaire a la vôtre avant d'enregistrez.
[2]==> Toujour dire Merçi a la bonne réponse.
1-Titre :
Permet de déterminer la thématique du bug.
exemples:
[1]==> J'ai un problème de connexion avec ma base de donnée.
[2]==> Ma fonction (promesse) ne capte pas les erreurs.
2-Language :
Permet d'identifier le language du post et est aussi utilisé comme recherche
exemples:
[1...]==> PHP, Java Script ,Java ect ...
NB : Veillez bien saisir le language normalement.
3-Détails :
Le plus important! détail permet aux autres de comprendre ce qui c'est passé ou plutôt ce que vous voulez faire, vous allez expliquez ce que vous tentez de faire et comment, ce qui permettra aux autres d'avoir une idée claire sur le post car lire le(s) code(s) d'autrui peut vite devenir un calvaire du coup avec le(s) détail(s) l'approche sera plus simple aux autres. Alors soyez clair et direct sur le(s) détail(s).
4-Code(s) :
Vous allez copié et collé le code, maximum trois(3) codes car il se peut que vous travailez avec plusieurs fichiers comme un fichier html pour recupérer les données un fichier php pour traiter les données donc en cas d'erreur il est préférable que vous collez tout ces deux fichiers.
Enfin vous enregistrez le post qui sera notifié aux membres de CIDAD et procéderons aux suggestions de solutions.

APP-PLUGIN

App-Pluging permet de partager des codes (script) prêt a l'emploi!
NB :
Le code d'un plugin ne doit faire une et une seule et doit bien la faire.
Ne codé pas un script qui existe déja pour les languages exemples : Un code qui trie un TB, Un code qui supprime les doublons d'un TB ect ...
Cherché a faire des scripts plus complexe, factorisé un code, des animations CSS ect ...
Partager le plugin sur [Enregistrer Plugin],le plugin aura un Language, Documentation et le Code.
Régle D'or :
[1]==> Toujour vérifier s'il n'y a pas un plugin similaire a la vôtre avant d'enregistrez.
1-Language :
Permet d'identifier le language du plugin et est aussi utilisé comme recherche
exemples:
[1...]==> PHP, Java Script ,Java ect ...
NB : Veillez bien saisir le language normalement.
2-Documentation :
Permet aux autres de ce documenté sur ce plugin et de savoir comment l'implémenté dans leur code, et aussi quel est le problème résolu par ce plugin, en quoi est t'il important.
3-Code :
Le code du plugin, vous allez copié et le collé.
Enfin vous enregistrez le plugin qui sera notifié aux membres de CIDAD qui pourra aidé certains d'entre eux lors de leur développement.

APP-URL-VIDEO

Partager les liens des tutorats! d'abord un contexte ils nous est arrivé parfois de faire cour sur Google meet pour des raisons quelconques, le cour est enregistré par le tuteur ensuite passe le lien de ce cour aux responsables qui a leur tour crée un compte gmail public y enregistre tout les liens(URL) pour que les autres puissent voir ou revoir le cour.
Bien !!!oui c'est bien mais il se trouve qu'il y a problème oui (Compte Gmail Public) tout le monde y avait accés ce qui est normal mais par erreur certains supprimé les liens(URL) ou les classés dans le spam, par conséquence les autres ne pourrons plus voir ou revoir ce cour.
App-URL-Video permet aux réponsables d'enregistré les liens(URL) des cours sur meet en toute sécurité seul celui qui la partagé peut le supprimé, les autres ne pourrons que visionnez, copié ou signalé le lien(URL). Partager le app-url-video sur [Enregistrer URL-Video], l' URL-Video aura nom-Cour, Licence-Séquence et l'URL.
Régles D'or :
[1]==> Toujour vérifier s'il n'y a pas un URL similaire a la vôtre avant d'enregistrez.
[2]==> Pas d'URL non sécurisé qui sera même rejeté.
[3]==> Pas d'URL YouTube qui sera même rejeté.
1-nom-Cour :
Permet d'identifier le cour exemples:
[1]==> Language C
[2]==> Language Java
[3]==> JSP et Servlet
NB : Veillez bien saisir le nom-Cour normalement.
2-Licence-Séquence :
A sélectionné exemples:
[1]==> LICENSE 1 SEMESTRE 1 [Séquence : 2]
[2]==> LICENSE 3 SEMESTRE 2 [Séquence : 4]
3-l'URL :
Le lien(URL) du cour, vous allez copié et le collé.
Enfin vous enregistrez le lien qui sera notifié aux membres de CIDAD qui permettra a certains de voir et revoir le cour sans problème.

TOKEN

Les tokens assurent la sécurité de vos requêtes vers l'API-REST et votre authentification qui est valable 72h. Si le token n'est plus valide ou cache nettoyer Vous serez notifié de ce token réjeté par l'API puis [Généré Token] va s'affiché.
Cliquer sur ce bouton puis saisir votre mot de passe et un nouveau token vous serez généré pour 72h.

MODE DE SECURITÉ

Définir votre mode de sécurité pour votre compte CIDAD dans paramètre N°05
1-Mode Sécurisé :
Une demande de code de confirmation sur toute les actions sensibles comme une suppression, modification ect ...Pour bien certifié que vous éte l'auteur de cette action demandé avec trois (3) tentatives. Ce qui protégera vos données au cas ou si un autre utilisé votre machine a votre insu et que vous vous ete déja connecté à CIDAD.
2-Mode Non Sécurisé :
Pas de demande de code de confirmation sur toute les actions sensibles comme une suppression, modification ect ...Ce qui par conséquence donnera la possibilité a un autre de supprimé vos données ou de les modifiées si et seulement si votre séssion est ouverte et qu'il est en possession de votre ordinateur ou téléphone connecté.

MODE DE NAVIGATION

Avec CIDAD vous avez la possibilité de naviguer en mode identifiable ou mode fantôme dans Edit Profil
1-Mode Identifiable :
Vos informations personnelles comme votre Nom, Prénom, Promotion et photo profil serrons presentées aux autres si demandé. Vos plublications serrons identifiées de même que vos réponses et commentaires.
2-Mode Fantôme :
Il peut arrivé qu'il est un utilisateur qui veut posté un bug qui lui semble gênant ou vouloir répondre, commenté sans que l'on l'identifie.
Avec Mode Fantôme activé, vos informations personnelles ne serrons pas presentées aux autres, vos plublications ne serrons pas identifiées de même que vos réponses et commentaires.

CODE

Avec CIDAD vous avez un code de quatre (4) chiffres si le Mode Sécurisé est activé
1-Modifier le code :
Dans paramètre N°01 saisir votre ancien code ensuite le nouveau code et enfin validé la modification
2-Réinitialiser le code :
Dans paramètre N°02 saisir votre mail ensuite mot de passe pour certifié que vous éte l'auteur de cette action de réinitialisation du code et enfin validé. Si vérification valide le code sera réinitialisé a :1234

MOT DE PASSE

Votre mot de passe vous permet de vous connecté à CIDAD
1-Modifier mot de passe :
Dans paramètre N°04 saisir votre ancien mot de passe ensuite le nouveau mot de passe et enfin validé la modification
2-Réinitialiser mot de passe :
Déconnectez-vous d'abord allez dans la page connexion cliquez sur mot de passe oublier, et de bien suivre les directives un mail de réinitialisation vous serez envoyé. Définir le nouveau mot de passe suivant ce mail envoyé puis revenir a la page de connexion pour enregistrez le nouveau mot de passe que vous avez défini, et vous serez automatiquement reconnecté.

MAIL

Votre mail vous permet de vous connecté à CIDAD de réinitialisé votre code, mot de passe ect ...
1-Modifier mail :
Dans paramètre N°03
saisir votre nouveau mail puis mot de passe et enfin validé la modification. La modification sera effectuée si et seulement si votre nouveau mail n'est pas utilisé par un autre.

NOTIFICATIONS

Recevez des notifications et alertes en temps réel des nouveaux posts, plugin, url-video, réponses et commentaires.
Vous serez notifié de tout ce qui ce passe dans vos posts et plugin. Si vous proposez une réponse a un post vous serez notifier des commentaires sur votre réponse, uniquement votre réponse.
Vous serez notifié que des commentaires qui vous concerne, si le post n'est pas le vôtre idem que la ou les réponses et que vous n'avez fait au commentaires donc vous ne recevrez aucune notifications concernent ce post.

ZONE RÉINITIALISATIONS

Vous avez la possibilité de définir des zones à réinitialisées, car il se peut que vous voulez supprimé uniquement vos post(s), ou réponse(s) ect ...
Dans paramètre N°06.
Sélectionner la zone a réinitialisée, saisir votre mot de passe enfin validé. Si mot de passe valide la réinitialisation sera lancée.

RÉINITIALISATION

Vous pouvez réinitialisé toute les données de votre compte CIDAD, repartir à zéro
Dans paramètre N°07.
Saisir votre mot de passe enfin validé. Si mot de passe valide la réinitialisation générale sera lancée.
